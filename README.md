## ✨ 核心功能

### 📖 百科查询
- ✅ **宠物查询**：名称、属性、六维、特性、技能列表、立绘图片
- ✅ **技能查询**：名称、属性、威力、效果描述、图标
- ✅ **道具查询**：名称、分类、稀有度、获取方式、图片
- ✅ **家具查询**：家园装饰摆件，支持颜色筛选
- ✅ **精灵蛋/咕噜球/果实/技能石**：各类游戏物品信息
- ✅ **颜色筛选**：按颜色查询宠物、精灵蛋、家具、道具等
- ✅ **属性克制**：自动计算属性相克倍率
- ✅ **模糊搜索**：支持关键词模糊匹配，智能推荐
- ✅ **自然语言**：支持"迪莫会什么技能"、"迪莫怎么进化"等口语化表达

### 🖼️ 图文回复
- ✅ 查询宠物时自动发送对应立绘图片
- ✅ 查询道具/家具时发送物品图片
- ✅ 支持本地图片缓存，提升响应速度
- ✅ 图片缺失时优雅降级为纯文本回复

### ⚙️ 高度可配置
- ✅ WebUI 可视化配置，无需修改代码
- ✅ 自定义查询指令名称（如 `/lk`、`/roco`）
- ✅ 灵活调整搜索限制、触发关键词等参数

### 🔐 管理员功能
- ✅ 在线更新数据库（从 Wiki 爬取最新数据）
- ✅ 查看数据库状态统计
- ✅ 批量识别颜色（大模型视觉识别，智能准确）
- ✅ 补全缺失数据（自动检测并修复六维和图片路径）
- ✅ 通过 AstrBot 权限系统控制访问，安全可靠

## 🚀 快速开始

### 前置要求
- Python ≥ 3.8
- AstrBot ≥ 2.0
- Pydantic ≥ 2.0
- requests, aiohttp, Pillow, openai

### 安装步骤

#### 1. 安装依赖
```bash
cd astrbot_plugin_roco_world_wiki_search
pip install -r requirements.txt
```

#### 2. 构建数据库
```bash
# 全量爬取（包含图片和完整数据）
python -m src.build_wiki_db build --full

# 或仅爬取基础数据（不含图片）
python -m src.build_wiki_db build
```

#### 3. 部署到 AstrBot
将插件文件夹复制到 AstrBot 的插件目录：
```
AstrBot/data/plugins/astrbot_plugin_roco_world_wiki_search/
```

#### 4. 启用插件
在 AstrBot WebUI 中启用插件，插件会自动加载数据库。

## ⚙️ 配置说明

### WebUI 配置项

| 配置项 | 默认值 | 类型 | 说明 |
|--------|--------|------|------|
| `db_path` | `./wiki-local.db` | String | 数据库文件路径 |
| `search_limit` | `5` | Integer | 搜索结果最大数量 (1-20) |
| `enable_fuzzy_search` | `true` | Boolean | 是否启用模糊搜索 |
| `response_style` | `简洁` | String | 响应风格：简洁/详细 |
| `trigger_keywords` | `["洛克王国", "查询", "百科"]` | List[str] | 触发查询的关键词 |
| `query_command` | `查询` | String | 查询指令名称（不含/前缀） |
| `page_size` | `10` | Integer | 列表查询每页显示数量 (5-30) |
| `vision_model_config` | `""` | String | 视觉模型配置（从 AstrBot 已配置的模型中选择） |
| `manual_vision_api_key` | `sk-lm-xxx` | String | 手动填写的 API Key（留空则使用 AstrBot 配置） |
| `manual_vision_base_url` | `http://localhost:1234/v1` | String | 手动填写的 API Base URL |
| `manual_vision_model_id` | `qwen-vl` | String | 手动填写的模型 ID |

### 配置示例

在 AstrBot WebUI 的插件配置页面中：

```json
{
  "db_path": "./wiki-local.db",
  "search_limit": 5,
  "enable_fuzzy_search": true,
  "response_style": "简洁",
  "trigger_keywords": ["洛克王国", "查询", "百科"],
  "query_command": "查询",
  "page_size": 10
}
```

## 🔧 开发指南

### 添加新的查询类型

1. 在 `main.py` 的 `_parse_type_query()` 方法中添加新的正则表达式模式
2. 实现对应的处理函数（如 `_handle_xxx_filter()`）
3. 在 `on_message()` 中调用新的处理逻辑

### 自定义响应风格

修改 `_format_pet_response()` 和 `_format_skill_response()` 方法，根据 `self.response_style` 返回不同格式的内容。

### 扩展颜色识别

编辑 `src/color_extractor_vision.py`，可以调整：
- API 配置（API Key、Base URL、Model ID）
- 提示词模板（优化颜色识别准确性）
- 颜色映射表（支持的中文颜色名称）
- 请求超时和重试策略

### 数据维护

#### 初次构建数据库
```bash
python -m src.build_wiki_db build --full
```

#### 增量更新数据
```bash
python -m src.incremental_update
```

#### 补全缺失数据
```bash
python tools/fix_missing_pet_data.py
```

或在 AstrBot 中使用管理员命令：
```bash
/洛克管理 fix-missing
```

## 🐛 常见问题

### Q1: 查询返回"未找到相关信息"

**解决方案**：
1. 检查数据库是否正确构建：`/洛克管理 status`
2. 尝试使用更完整的名称
3. 启用模糊搜索：在 WebUI 中设置 `enable_fuzzy_search = true`

### Q2: 图片无法显示

**解决方案**：
1. 检查图片路径是否正确
2. 确认 `output/images/` 目录存在且有图片
3. 重新运行 `python -m src.build_wiki_db build --full`

### Q3: 管理员命令提示权限不足

**解决方案**：
1. 在 AstrBot WebUI 的权限管理中确认用户具有管理员权限
2. 使用正确的命令格式：`/洛克管理 update`
3. 联系服务器管理员分配权限

### Q4: 颜色查询没有结果

**解决方案**：
1. 运行 `/洛克管理 tag-colors` 识别家具/道具颜色
2. 运行 `/洛克管理 tag-pet-colors` 识别宠物颜色
3. 确认 WebUI 中配置了正确的视觉模型 API 信息
4. 等待识别完成后再次查询

### Q5: 部分宠物缺少六维数据或图片

**解决方案**：
1. 运行 `/洛克管理 fix-missing` 自动补全缺失数据
2. 或手动运行 `python tools/fix_missing_pet_data.py`
3. 如果仍然缺失，可能是原始 Wiki 页面就没有这些数据
