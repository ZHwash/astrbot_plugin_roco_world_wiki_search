# AstrBot 开发文档

## 插件开发指南 🌠

发布时间：2026-02-13 13:12:00

### 前置要求

- 有一定的 Python 编程经验

- 有一定的 Git、GitHub 使用经验

- 开发者专用 QQ 群: `975206796`

### 环境准备

#### 获取插件模板

1. 打开 AstrBot 插件模板: helloworld

2. 点击右上角的 `Use this template`

3. 点击 `Create new repository`

4. 在 `Repository name` 处填写插件名，要求：

    - 推荐以 `astrbot_plugin_` 开头

    - 不能包含空格

    - 保持全部字母小写

    - 尽量简短

5. 点击右下角的 `Create repository`

#### 克隆项目到本地

```Bash

git clone https://github.com/AstrBotDevs/AstrBot
mkdir -p AstrBot/data/plugins
cd AstrBot/data/plugins
git clone 插件仓库地址
```

- 使用 `VSCode` 打开 `AstrBot` 项目，找到 `data/plugins/<你的插件名字>` 目录

- **必须更新** `metadata.yaml` 文件，填写插件元数据信息（AstrBot 依赖此文件识别插件）

#### 设置插件 Logo（可选）

- 在插件目录下添加 `logo.png` 文件

- 长宽比 1:1，推荐尺寸 256x256

#### 插件展示名（可选）

- 修改/添加 `metadata.yaml` 文件中的 `display_name` 字段，作为插件市场展示名

### 调试插件

- AstrBot 采用运行时注入插件机制，调试需启动 AstrBot 本体

- 支持热重载：代码修改后，在 WebUI 插件管理处找到插件，点击右上角 `...` 选择 `重载插件`

### 插件依赖管理

- 使用 `pip` 自带的 `requirements.txt` 文件管理依赖

- 插件依赖第三方库时，务必在插件目录下创建该文件并写入依赖，避免用户安装时出现 `Module Not Found` 错误

- 格式参考 pip 官方文档

### 开发原则

1. 功能需经过测试

2. 需包含良好的注释

3. 持久化数据存储于 `data` 目录，而非插件自身目录，防止更新/重装插件时数据被覆盖

4. 实现良好的错误处理机制，避免插件因单个错误崩溃

5. 提交前使用 ruff 工具格式化代码

6. 网络请求使用 `aiohttp`、`httpx` 等异步库，禁止使用 `requests`

7. 功能扩增优先向原插件提交 PR，除非原插件作者停止维护

## 最小实例

发布时间：2026-02-13 13:12:00

插件模版中的 `main.py` 是最小插件实例：

```Python

from astrbot.api.event import filter, AstrMessageEvent, MessageEventResult
from astrbot.api.star import Context, Star, register
from astrbot.api import logger # 使用 astrbot 提供的 logger 接口

@register("helloworld", "author", "一个简单的 Hello World 插件", "1.0.0", "repo url")
class MyPlugin(Star):
    def __init__(self, context: Context):
        super().__init__(context)

    # 注册指令的装饰器。指令名为 helloworld。注册成功后，发送 `/helloworld` 就会触发这个指令，并回复 `你好, {user_name}!`
    @filter.command("helloworld")
    async def helloworld(self, event: AstrMessageEvent):
        '''这是一个 hello world 指令''' # 这是 handler 的描述，将会被解析方便用户了解插件内容。非常建议填写。
        user_name = event.get_sender_name()
        message_str = event.message_str # 获取消息的纯文本内容
        logger.info("触发hello world指令!")
        yield event.plain_result(f"Hello, {user_name}!") # 发送一条纯文本消息

    async def terminate(self):
        '''可选择实现 terminate 函数，当插件被卸载/停用时会调用。'''
```

### 核心说明

1. 插件必须继承 `Star` 类

2. `Context` 类用于插件与 AstrBot Core 交互，调用核心 API

3. 处理函数 `Handler` 需在插件类中定义，第一个两个参数必须为 `self` 和 `event`

4. `AstrMessageEvent` 是消息事件对象，存储发送者、消息内容等信息

5. `AstrBotMessage` 是消息对象，可通过 `event.message_obj` 获取

6. 插件类所在文件必须命名为 `main.py`

7. 所有处理函数需写在插件类中，文件行数过长时可将服务写在外部并在 `Handler` 中调用

## 消息与事件处理

发布时间：2026-02-13 13:12:00

### 核心导入

```Python

from astrbot.api.event import filter, AstrMessageEvent
```

**注意**：必须导入，否则会与 Python 高阶函数 `filter` 冲突

### 消息与事件基础

- AstrBot 将平台消息封装为 `AstrMessageEvent` 对象传递给插件

- `AstrBotMessage` 是消息对象，存储平台消息具体内容，通过 `event.message_obj` 获取

#### AstrBotMessage 结构

```Python

class AstrBotMessage:
    '''AstrBot 的消息对象'''
    type: MessageType  # 消息类型
    self_id: str  # 机器人的识别id
    session_id: str  # 会话id。取决于 unique_session 的设置。
    message_id: str  # 消息id
    group_id: str = "" # 群组id，如果为私聊，则为空
    sender: MessageMember  # 发送者
    message: List[BaseMessageComponent]  # 消息链。比如 [Plain("Hello"), At(qq=123456)]
    message_str: str  # 最直观的纯文本消息字符串，将消息链中的 Plain 消息（文本消息）连接起来
    raw_message: object
    timestamp: int  # 消息时间戳
```

- `raw_message`：消息平台适配器的原始消息对象

### 消息链

- 描述消息结构的有序列表，每个元素为**消息段**

- 类型为 `List[BaseMessageComponent]`

#### 通用消息段类型

- `Plain`：文本消息段

- `At`：提及消息段

- `Image`：图片消息段

- `Record`：语音消息段

- `Video`：视频消息段

- `File`：文件消息段

#### OneBot v11 额外支持

- `Face`：表情消息段

- `Node`：合并转发消息单个节点

- `Nodes`：合并转发消息多个节点

- `Poke`：戳一戳消息段

### 指令

#### 基础指令

```Python

from astrbot.api.event import filter, AstrMessageEvent
from astrbot.api.star import Context, Star, register

@register("helloworld", "Soulter", "一个简单的 Hello World 插件", "1.0.0")
class MyPlugin(Star):
    def __init__(self, context: Context):
        super().__init__(context)

    @filter.command("helloworld")
    async def helloworld(self, event: AstrMessageEvent):
        '''这是 hello world 指令'''
        user_name = event.get_sender_name()
        yield event.plain_result(f"Hello, {user_name}!")
```

**注意**：指令名不能带空格，否则会被解析为第二个参数

#### 带参指令

AstrBot 自动解析指令参数：

```Python

@filter.command("add")
def add(self, event: AstrMessageEvent, a: int, b: int):
    # /add 1 2 -> 结果是: 3
    yield event.plain_result(f"Wow! The anwser is {a + b}!")
```

#### 指令组

用于组织相关指令，支持无限嵌套：

```Python

# 一级指令组
@filter.command_group("math")
def math(self):
    pass

@math.command("add")
async def add(self, event: AstrMessageEvent, a: int, b: int):
    yield event.plain_result(f"结果是: {a + b}")

@math.command("sub")
async def sub(self, event: AstrMessageEvent, a: int, b: int):
    yield event.plain_result(f"结果是: {a - b}")

# 嵌套指令组
@math.group("calc") # 嵌套使用 group 而非 command_group
def calc():
    pass

@calc.command("add")
async def add(self, event: AstrMessageEvent, a: int, b: int):
    yield event.plain_result(f"结果是: {a + b}")
```

- 指令组函数内直接 `pass` 即可

- 用户未输入子指令时，会报错并渲染指令组树形结构

#### 指令别名（v3.4.28+）

为指令/指令组添加多个别名：

```Python

@filter.command("help", alias={'帮助', 'helpme'})
def help(self, event: AstrMessageEvent):
    yield event.plain_result("这是一个计算器插件，拥有 add, sub 指令。")
```

### 事件类型过滤

#### 接收所有消息

```Python

@filter.event_message_type(filter.EventMessageType.ALL)
async def on_all_message(self, event: AstrMessageEvent):
    yield event.plain_result("收到了一条消息。")
```

#### 区分群聊/私聊

```Python

@filter.event_message_type(filter.EventMessageType.PRIVATE_MESSAGE)
async def on_private_message(self, event: AstrMessageEvent):
    message_str = event.message_str
    yield event.plain_result("收到了一条私聊消息。")
```

- `EventMessageType` 为 Enum 类型，包含 `PRIVATE_MESSAGE`、`GROUP_MESSAGE`

#### 消息平台过滤

```Python

@filter.platform_adapter_type(filter.PlatformAdapterType.AIOCQHTTP | filter.PlatformAdapterType.QQOFFICIAL)
async def on_aiocqhttp(self, event: AstrMessageEvent):
    '''只接收 AIOCQHTTP 和 QQOFFICIAL 的消息'''
    yield event.plain_result("收到了一条信息")
```

- `PlatformAdapterType` 包含 `AIOCQHTTP`、`QQOFFICIAL`、`GEWECHAT`、`ALL`

#### 管理员指令

```Python

@filter.permission_type(filter.PermissionType.ADMIN)
@filter.command("test")
async def test(self, event: AstrMessageEvent):
    pass
```

- 仅管理员可使用该指令

#### 多个过滤器

多个装饰器同时使用，遵循 **AND** 逻辑：

```Python

@filter.command("helloworld")
@filter.event_message_type(filter.EventMessageType.PRIVATE_MESSAGE)
async def helloworld(self, event: AstrMessageEvent):
    yield event.plain_result("你好！")
```

### 事件钩子

**注意**：事件钩子不与 `@filter.command`/`command_group`/`event_message_type` 等装饰器一起使用

#### Bot 初始化完成时（v3.4.34+）

```Python

from astrbot.api.event import filter, AstrMessageEvent

@filter.on_astrbot_loaded()
async def on_astrbot_loaded(self):
    print("AstrBot 初始化完成")
```

#### 等待 LLM 请求时

```Python

from astrbot.api.event import filter, AstrMessageEvent

@filter.on_waiting_llm_request()
async def on_waiting_llm(self, event: AstrMessageEvent):
    await event.send("🤔 正在等待请求...")
```

- 不可使用 `yield` 发送消息，需用 `event.send()`

#### LLM 请求时

可修改 `ProviderRequest` 对象（包含请求文本、系统提示等）：

```Python

from astrbot.api.event import filter, AstrMessageEvent
from astrbot.api.provider import ProviderRequest

@filter.on_llm_request()
async def my_custom_hook_1(self, event: AstrMessageEvent, req: ProviderRequest):
    print(req)
    req.system_prompt += "自定义 system_prompt"
```

- 不可使用 `yield` 发送消息，需用 `event.send()`

#### LLM 请求完成时

可修改 `ProviderResponse` 对象：

```Python

from astrbot.api.event import filter, AstrMessageEvent
from astrbot.api.provider import LLMResponse

@filter.on_llm_response()
async def on_llm_resp(self, event: AstrMessageEvent, resp: LLMResponse):
    print(resp)
```

- 不可使用 `yield` 发送消息，需用 `event.send()`

#### 发送消息前

装饰消息内容，如加前缀、转格式：

```Python

from astrbot.api.event import filter, AstrMessageEvent

@filter.on_decorating_result()
async def on_decorating_result(self, event: AstrMessageEvent):
    result = event.get_result()
    chain = result.chain
    print(chain)
    chain.append(Plain("!")) # 消息链末尾添加感叹号
```

- 仅装饰 `event.get_result().chain`，不可用 `yield` 发消息

#### 发送消息后

```Python

from astrbot.api.event import filter, AstrMessageEvent

@filter.after_message_sent()
async def after_message_sent(self, event: AstrMessageEvent):
    pass
```

- 不可使用 `yield` 发送消息，需用 `event.send()`

### 优先级

指令、监听器、钩子可设置优先级，默认 `0`，数值越高执行越靠前：

```Python

@filter.command("helloworld", priority=1)
async def helloworld(self, event: AstrMessageEvent):
    yield event.plain_result("Hello!")
```

### 控制事件传播

停止事件传播后，后续所有操作（如其他插件处理、LLM 请求）均不执行：

```Python

@filter.command("check_ok")
async def check_ok(self, event: AstrMessageEvent):
    ok = self.check()
    if not ok:
        yield event.plain_result("检查失败")
        event.stop_event() # 停止事件传播
```

## 消息的发送

发布时间：2026-02-13 13:12:00

### 被动消息

机器人被动回复消息，支持多段发送：

```Python

@filter.command("helloworld")
async def helloworld(self, event: AstrMessageEvent):
    yield event.plain_result("Hello!")
    yield event.plain_result("你好！")

    yield event.image_result("path/to/image.jpg") # 本地图片
    yield event.image_result("https://example.com/image.jpg") # 网络图片（必须http/https开头）
```

### 主动消息

机器人主动推送消息，需通过 `unified_msg_origin` 定位会话：

```Python

from astrbot.api.event import MessageChain

@filter.command("helloworld")
async def helloworld(self, event: AstrMessageEvent):
    umo = event.unified_msg_origin
    message_chain = MessageChain().message("Hello!").file_image("path/to/image.jpg")
    await self.context.send_message(event.unified_msg_origin, message_chain)
```

- `unified_msg_origin`：会话唯一ID，格式为`platform_name:message_type:session_id`

- 可存储该ID，后续按需发送消息

### 富媒体消息

使用 `MessageChain` 构建包含多种类型的消息链：

```Python

import astrbot.api.message_components as Comp

@filter.command("helloworld")
async def helloworld(self, event: AstrMessageEvent):
    chain = [
        Comp.At(qq=event.get_sender_id()), # At发送者
        Comp.Plain("来看这个图："),
        Comp.Image.fromURL("https://example.com/image.jpg"), # 网络图片
        Comp.Image.fromFileSystem("path/to/image.jpg"), # 本地图片
        Comp.Plain("这是一个图片。")
    ]
    yield event.chain_result(chain)
```

#### 其他富媒体类型

##### 文件

```Python

Comp.File(file="path/to/file.txt", name="file.txt") # 部分平台不支持
```

##### 语音

```Python

path = "path/to/record.wav" # 仅支持wav格式
Comp.Record(file=path, url=path)
```

##### 视频

```Python

path = "path/to/video.mp4"
Comp.Video.fromFileSystem(path=path) # 本地视频
Comp.Video.fromURL(url="https://example.com/video.mp4") # 网络视频
```

#### 注意

- aiocqhttp 适配器中，`Plain` 消息会自动 `strip()` 空格/换行，可添加零宽空格 `\u200b` 解决

### 发送视频消息

```Python

from astrbot.api.event import filter, AstrMessageEvent

@filter.command("test")
async def test(self, event: AstrMessageEvent):
    from astrbot.api.message_components import Video
    # 本地视频（协议端与机器人端同系统）

    # 网络视频（通用）
    music = Video.fromURL(url="https://example.com/video.mp4")
```


## 插件配置

发布时间：2026-02-13 13:12:00

### 配置定义

在插件目录下创建 `_conf_schema.json` 文件，通过 Schema 定义配置，支持可视化配置面板：

#### 基础 Schema 示例

```JSON

{
  "token": {
    "description": "Bot Token",
    "type": "string"
  },
  "sub_config": {
    "description": "测试嵌套配置",
    "type": "object",
    "hint": "xxxx",
    "items": {
      "name": {
        "description": "testsub",
        "type": "string",
        "hint": "xxxx"
      },
      "id": {
        "description": "testsub",
        "type": "int",
        "hint": "xxxx"
      },
      "time": {
        "description": "testsub",
        "type": "int",
        "hint": "xxxx",
        "default": 123
      }
    }
  }
}
```

#### Schema 字段说明

|字段|必选|说明|
|---|---|---|
|type|是|配置类型：string/text/int/float/bool/object/list/dict/template_list/file|
|description|否|配置描述，建议一句话说明|
|hint|否|配置提示，鼠标悬浮在面板问号按钮时显示|
|obvious_hint|否|是否醒目显示 hint|
|default|否|默认值，未配置时使用（int=0/float=0.0/bool=False/string=""）|
|items|否|object类型的子Schema，支持无限嵌套|
|invisible|否|是否在面板隐藏，默认false|
|options|否|下拉列表可选项，如["chat", "agent"]|
|editor_mode|否|是否启用代码编辑器（AstrBot>=v3.5.10），默认false|
|editor_language|否|代码编辑器语言，默认json|
|editor_theme|否|编辑器主题：vs-light/vs-dark，默认vs-light|
|_special|否|v4.0.0+，支持select_provider/select_persona等可视化选择|
#### 特殊类型 Schema

##### file 类型（v4.13.0+）

支持用户上传文件：

```JSON

{
  "demo_files": {
    "type": "file",
    "description": "Uploaded files for demo",
    "default": [],
    "file_types": ["pdf", "docx"] // 允许的文件类型
  }
}
```

##### dict 类型

可视化编辑Python字典配置，支持滑块：

```JSON

"custom_extra_body": {
  "description": "自定义请求体参数",
  "type": "dict",
  "items": {},
  "hint": "添加temperature、top_p等参数",
  "template_schema": {
      "temperature": {
          "name": "Temperature",
          "description": "温度参数",
          "type": "float",
          "default": 0.6,
          "slider": {"min": 0, "max": 2, "step": 0.1}
      }
  }
}
```

##### template_list 类型（v4.10.4+）

嵌套模板配置：

```JSON

"field_id": {
  "type": "template_list",
  "description": "Template List Field",
  "templates": {
    "template_1": {
        "name": "Template One",
        "hint":"hint",
        "items": {
          "attr_a": {
            "description": "Attribute A",
            "type": "int",
            "default": 10
          }
        }
      }
  }
}
```

**保存后配置格式**：

```JSON

"field_id": [
    {
        "__template_key": "template_1",
        "attr_a": 10
    }
]
```

### 在插件中使用配置

AstrBot 自动解析 `_conf_schema.json`，生成配置文件到 `data/config/<plugin_name>_config.json`，并传入插件初始化方法：

```Python

from astrbot.api import AstrBotConfig
from astrbot.api.star import Context, Star, register

@register("config", "Soulter", "一个配置示例", "1.0.0")
class ConfigPlugin(Star):
    def __init__(self, context: Context, config: AstrBotConfig): # AstrBotConfig 继承Dict
        super().__init__(context)
        self.config = config
        print(self.config)
        # 保存配置
        # self.config.save_config()
```

发布插件新版本更新 Schema 时，AstrBot 会：

1. 递归检查配置项

2. 自动为缺失项添加默认值

3. 移除不存在的配置项

### 定义 Tool

大模型调用外部工具的能力，支持两种定义方式：

#### 方式1：基于 dataclass

```Python

from pydantic import Field
from pydantic.dataclasses import dataclass

from astrbot.core.agent.run_context import ContextWrapper
from astrbot.core.agent.tool import FunctionTool, ToolExecResult
from astrbot.core.astr_agent_context import AstrAgentContext

@dataclass
class BilibiliTool(FunctionTool[AstrAgentContext]):
    name: str = "bilibili_videos"  # 工具名称
    description: str = "A tool to fetch Bilibili videos."  # 工具描述
```

### 基础示例（成语接龙）

```Python

from astrbot.api.event import filter, AstrMessageEvent

@filter.command("成语接龙")
async def handle_empty_mention(self, event: AstrMessageEvent):
    """成语接龙具体实现"""
    try:
        yield event.plain_result("请发送一个成语~")

        # 注册会话控制器，超时60秒，不记录历史消息链
        @session_waiter(timeout=60, record_history_chains=False)
        async def empty_mention_waiter(controller: SessionController, event: AstrMessageEvent):
            idiom = event.message_str

            if idiom == "退出":
                await event.send(event.plain_result("已退出成语接龙~"))
                controller.stop() # 停止会话
                return

            if len(idiom) != 4:
                await event.send(event.plain_result("成语必须是四个字的呢~"))
                return # 不停止会话，继续等待

            # 发送接龙结果
            message_result = event.make_result()
            message_result.chain = [Comp.Plain("先见之明")]
            await event.send(message_result)

            # 重置超时时间，继续会话
            controller.keep(timeout=60, reset_timeout=True)

        try:
            await empty_mention_waiter(event)
        except TimeoutError as _:
            yield event.plain_result("你超时了！")
        except Exception as e:
            yield event.plain_result("发生错误，请联系管理员: " + str(e))
        finally:
            event.stop_event()
    except Exception as e:
        logger.error("handle_empty_mention error: " + str(e))
```

### SessionController 方法

|方法|功能|
|---|---|
|keep(timeout, reset_timeout)|保持会话，reset_timeout=True则重置超时|
|stop()|立即结束会话|
|get_history_chains()|获取历史消息链（需开启record_history_chains）|
### 自定义会话ID算子

```Python

import asyncio

from astrbot.api.platform import Platform, AstrBotMessage, MessageMember, PlatformMetadata, MessageType
from astrbot.api.event import MessageChain
from astrbot.api.message_components import Plain, Image, Record
from astrbot.core.platform.astr_message_event import MessageSesion
from astrbot.api.platform import register_platform_adapter
from astrbot import logger
from .client import FakeClient
from .fake_platform_event import FakePlatformEvent
            
# 注册平台适配器：平台名、描述、默认配置
@register_platform_adapter("fake", "fake 适配器", default_config_tmpl={
    "token": "your_token",
    "username": "bot_username"
})
class FakePlatformAdapter(Platform):

    def __init__(self, platform_config: dict, platform_settings: dict, event_queue: asyncio.Queue) -> None:
        super().__init__(event_queue)
        self.config = platform_config # 用户填写的配置
        self.settings = platform_settings # 平台全局设置
    
    async def send_by_session(self, session: MessageSesion, message_chain: MessageChain):
        # 必须实现：按会话发送消息
        await super().send_by_session(session, message_chain)
    
    def meta(self) -> PlatformMetadata:
        # 必须实现：返回平台元数据
        return PlatformMetadata(
            "fake",
            "fake 适配器",
        )

    async def run(self):
        # 必须实现：适配器主逻辑
        async def on_received(data):
            logger.info(data)
            abm = await self.convert_message(data=data) # 转换为AstrBotMessage
            await self.handle_msg(abm) 
        
        # 初始化客户端并启动监听
        self.client = FakeClient(self.config['token'], self.config['username'])
        self.client.on_message_received = on_received
        await self.client.start_polling() # 阻塞监听消息

    async def convert_message(self, data: dict) -> AstrBotMessage:
        # 必须实现：平台消息 -> AstrBotMessage
        abm = AstrBotMessage()
        abm.type = MessageType.GROUP_MESSAGE # 消息类型：GROUP_MESSAGE/PRIVATE_MESSAGE
        abm.group_id = data['group_id'] # 群ID，私聊留空
        abm.message_str = data['content'] # 纯文本消息
        abm.sender = MessageMember(user_id=data['userid'], nickname=data['username']) # 发送者
        abm.message = [Plain(text=data['content'])] # 消息链
        abm.raw_message = data # 原始消息
        abm.self_id = data['bot_id'] # 机器人ID
        abm.session_id = data['userid'] # 会话ID
        abm.message_id = data['message_id'] # 消息ID
        
        return abm
    
    async def handle_msg(self, message: AstrBotMessage):
        # 处理消息并提交到事件队列
        message_event = FakePlatformEvent(
            message_str=message.message_str,
            message_obj=message,
            platform_meta=self.meta(),
            session_id=message.session_id,
            client=self.client
        )
        self.commit_event(message_event) # 提交事件，不可省略
```

### 步骤3：实现平台事件（[fake_platform_event.py](fake_platform_event.py)）

```Python

from astrbot.api.event import AstrMessageEvent, MessageChain
from astrbot.api.platform import AstrBotMessage, PlatformMetadata
from astrbot.api.message_components import Plain, Image
from .client import FakeClient
from astrbot.core.utils.io import download_image_by_url

class FakePlatformEvent(AstrMessageEvent):
    def __init__(self, message_str: str, message_obj: AstrBotMessage, platform_meta: PlatformMetadata, session_id: str, client: FakeClient):
        super().__init__(message_str, message_obj, platform_meta, session_id)
        self.client = client # 平台客户端实例
        
    async def send(self, message: MessageChain):
        # 实现消息发送逻辑，遍历消息链
        for i in message.chain:
            if isinstance(i, Plain): # 文本消息
                await self.client.send_text(to=self.get_sender_id(), message=i.text)
            elif isinstance(i, Image): # 图片消息
                img_url = i.file
                img_path = ""
                if img_url.startswith("file:///"):
                    img_path = img_url[8:]
                elif i.file and i.file.startswith("http"):
                    img_path = await download_image_by_url(i.file)
                else:
                    img_path = img_url
                await self.client.send_image(to=self.get_sender_id(), image_path=img_path)

        await super().send(message) # 必须调用父类send方法
```

## AstrBot HTTP API

发布时间：2026-03-26 12:03:00

**说明**：AstrBot>=v4.18.0 提供基于API Key的HTTP API，支持调用核心能力

### 快速开始

1. 在 WebUI - **设置** 中创建 **API Key**

2. 请求头携带API Key（二选一）：

    ```HTTP
    
    Authorization: Bearer abk_xxx
    # 或
    X-API-Key: abk_xxx
    ```

3. 对话接口**必须**传`username`参数

### Scope 权限说明

创建API Key时配置权限范围，未包含则返回`403 Insufficient API key scope`：

|Scope|作用|可访问接口|
|---|---|---|
|chat|对话能力、查询会话|POST /api/v1/chat、GET /api/v1/chat/sessions|
|config|获取配置文件列表|GET /api/v1/configs|
|file|上传附件|POST /api/v1/file|
|im|主动发IM消息、查询bot列表|POST /api/v1/im/message、GET /api/v1/im/bots|
### 常用接口

#### 对话类

- **POST /api/v1/chat**：发送对话消息，SSE流式返回，不传`session_id`自动创建UUID

- **GET /api/v1/chat/sessions**：分页获取指定`username`的会话列表

- **GET /api/v1/configs**：获取可用配置文件列表

#### 文件上传

- **POST /api/v1/file**：上传附件，返回`attachment_id`（用于富媒体消息）

#### IM消息发送

- **POST /api/v1/im/message**：按UMO主动发送消息到指定会话

- **GET /api/v1/im/bots**：获取bot/platform ID列表

### message 字段格式（重点）

`POST /api/v1/chat` 和 `POST /api/v1/im/message` 的`message`字段支持两种格式：

#### 1. 纯文本格式

```JSON

{
  "message": "Hello"
}
```

#### 2. 消息段数组格式（消息链）

```JSON

{
  "message": [
    { "type": "plain", "text": "请看这个文件" },
    { "type": "file", "attachment_id": "9a2f8c72-e7af-4c0e-b352-111111111111" }
  ]
}
```

##### 支持的消息段类型

|type|必填字段|可选字段|说明|
|---|---|---|---|
|plain|text|-|文本段|
|reply|message_id|selected_text|引用回复（仅/api/v1/chat支持）|
|image|attachment_id|-|图片附件|
|record|attachment_id|-|音频附件|
|file|attachment_id|-|通用文件|
|video|attachment_id|-|视频附件|
|**注意**：||||
- `attachment_id` 来自`POST /api/v1/file`上传结果

- `reply` 不能单独使用，需配合其他消息段

### 接口示例

#### 对话接口示例

```Bash

curl -N 'http://localhost:6185/api/v1/chat' \
  -H 'Authorization: Bearer abk_xxx' \
  -H 'Content-Type: application/json' \
  -d '{"message":"Hello","username":"alice"}'
```

**JSON请求体**：

```JSON

{
  "username": "alice",
  "session_id": "my_session_001",
  "message": [
    { "type": "plain", "text": "帮我总结这个PDF" },
    { "type": "file", "attachment_id": "9a2f8c72-e7af-4c0e-b352-111111111111" }
  ],
  "enable_streaming": true
}
```

#### IM消息发送示例

```JSON

{
  "umo": "webchat:FriendMessage:openapi_probe",
  "message": [
    { "type": "plain", "text": "这是主动消息" },
    { "type": "image", "attachment_id": "9a2f8c72-e7af-4c0e-b352-222222222222" }
  ]
}
```

### 核心字段详解

#### 基础字段

- `config_version`：配置版本，**禁止修改**

- `wake_prefix`：机器人唤醒前缀，默认`["/"]`

- `log_level`：日志级别，可选DEBUG/INFO/WARNING/ERROR/CRITICAL

- `timezone`：时区，填写IANA名称（如Asia/Shanghai）

- `plugin_set`：启用的插件列表，`["*"]`表示所有插件

#### platform_settings：平台通用设置

- `unique_session`：是否会话隔离（群内每人独立上下文）

- `rate_limit`：消息限流，`stall`等待/`discard`丢弃

- `enable_id_white_list`：是否启用ID白名单，仅白名单用户可触发

- `segmented_reply`：分段回复设置，支持随机/对数间隔

- `empty_mention_waiting`：空@机器人时等待后续消息合并处理

#### provider_settings：LLM提供商设置

- `enable`：是否启用LLM聊天

- `default_provider_id`：默认LLM提供商ID

- `web_search`：是否启用网页搜索能力

- `streaming_response`：是否启用流式响应（打字机效果）

- `max_agent_step`：Agent最大工具调用步骤

#### content_safety：内容安全

- `internal_keywords`：内部关键词过滤，支持正则

- `baidu_aip`：百度AI内容审核，需填写app_id/api_key/secret_key

#### dashboard：WebUI配置

- `enable`：是否启用WebUI

- `host/port`：监听地址和端口，默认[0.0.0.0:6185](0.0.0.0:6185)

- `username/password`：登录账号，密码为MD5加密值（**请勿直接修改**）

#### t2i：文本转图片

- `enable`：是否启用长文本转图片

- `t2i_word_threshold`：触发阈值，默认150字

- `t2i_strategy`：渲染策略，`local`本地/`remote`远程
